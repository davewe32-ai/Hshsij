Android Application Technical Specs

This portal hosts the verified master code structure of the Glorious Homes offline-first Android terminals.

Verified Package Architecture

The Android application runs on a clean architecture model matching this structure:

app/src/main/java/com/example/
MainActivity.kt (Master Controller Screen Router)
data/
dao/ (Room DAOs for relational queries)
database/ (AppDataBase schema with migrations)
entity/ (Material, Cabinet, Project, StockMovement models)
repository/ (Local data synchronization patterns)
ui/
screens/ (Jetpack Compose declarative views)
├── DashboardScreen.kt
├── MaterialsScreen.kt (Cabinet storage grids)
├── CabinetsScreen.kt (Rack layout tracking)
├── ProjectsScreen.kt (Construction sites tracker)
├── ReportsScreen.kt (CSV & PDF generation)
└── BackupRestoreScreen.kt (Security roles & settings)
viewmodel/ (InventoryViewModel StateFlow engine)

Modern Gradle & Build Specifications
